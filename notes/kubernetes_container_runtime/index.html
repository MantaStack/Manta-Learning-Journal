📋 架构说明
Kubernetes：声明式编排，通过 RuntimeClass 决定走 runc 或 kata 路径
kubelet：唯一与容器运行时交互的 K8s 组件，实现 CRI 客户端
containerd：枢纽层，根据 RuntimeClass 分发到 runc 或 kata-shim
runc vs Kata：runc 共享内核；Kata 为每个 Pod 创建独立 VM 沙箱
VMM 层：QEMU 功能最全；Cloud Hypervisor 最轻量；Firecracker 极简
KVM：所有 VMM 的硬件加速底座（Cloud Hypervisor 也可对接 MSHV）
MicroVM：Kata 启动的轻量虚拟机，内含独立内核 + kata-agent
Hardware：CPU 虚拟化扩展（Intel VT-x / AMD-V）是一切硬件加速的前提

Kubernetes 容器运行时全栈架构 | 基于 K8s 1.30+ / containerd 1.7+ 整理
