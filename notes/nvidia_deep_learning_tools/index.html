📦 核心组件详解

Hardware Interface
🔌
NVIDIA Driver
作用：操作系统与 GPU 的桥梁。管理 GPU 硬件资源、内存分配、任务调度。

关键：这是唯一必须安装在宿主机上的组件。所有 CUDA 程序都通过驱动与 GPU 通信。
宿主机安装 必须
Container Support
🐳
NVIDIA Container Toolkit
作用：让 Docker 容器能够访问宿主机的 GPU。将驱动文件和 GPU 设备映射进容器。

关键：安装后在 Docker 中可以使用 --gpus all 参数。宿主机安装即可，容器内不需要。
宿主机安装 docker --gpus
Development Platform
⚙️
CUDA Toolkit
作用：完整的 CUDA 开发环境。包含编译器(nvcc)、运行时库、调试工具、数学库等。

关键：通常安装在容器内，与宿主机驱动解耦。不同项目可用不同 CUDA 版本。
容器内安装 nvcc 开发必备
Runtime Only
▶️
CUDA Runtime
作用：CUDA 程序运行所需的最低环境，不含编译工具。

关键：如果你只运行预编译好的 CUDA 程序（如 PyTorch 二进制包），只需要 Runtime，不需要完整 Toolkit。
体积更小 运行即可
Deep Learning Primitives
🚀
cuDNN
作用：CUDA Deep Neural Network library。针对深度学习优化的 GPU 加速原语（卷积、RNN、注意力等）。

关键：PyTorch/TensorFlow 底层调用 cuDNN 实现 GPU 加速。必须与 CUDA 版本严格匹配。
框架依赖 版本敏感
Linear Algebra
🔢
cuBLAS / cuFFT / cuRAND
作用：CUDA 基础数学库。cuBLAS（线性代数）、cuFFT（快速傅里叶变换）、cuRAND（随机数生成）。

关键：PyTorch 的矩阵运算、TensorFlow 的卷积底层都依赖这些库。
CUDA Toolkit 包含 基础数学
Multi-GPU Communication
🔗
NCCL
作用：NVIDIA Collective Communications Library。多 GPU / 多节点间的集合通信（AllReduce、Broadcast 等）。

关键：分布式训练（DDP、DeepSpeed、Horovod）的核心依赖。支持 NVLink 和 InfiniBand 高速互联。
分布式训练 多卡必备
Inference Optimization
⚡
TensorRT
作用：推理优化引擎。通过层融合、精度校准（FP16/INT8）、内核自动调优，最大化推理性能。

关键：仅用于推理阶段，不用于训练。可将模型延迟降低 10-100 倍。
推理专用 性能优化
LLM Training
🤖
TransformerEngine
作用：NVIDIA 专为 Transformer 模型设计的加速库。提供 FP8 训练、FlashAttention 集成等。

关键：大语言模型（LLM）训练的重要加速组件，集成在最新 NGC PyTorch 镜像中。
LLM 专用 FP8

🔗 依赖关系链

1. NVIDIA Driver（宿主机）

必须最先安装。决定系统支持的最高 CUDA 版本。用 nvidia-smi 查看驱动版本和 GPU 状态。

⬇️

2. NVIDIA Container Toolkit（宿主机）

可选但强烈建议。让 Docker 容器能使用 GPU。安装后 docker run --gpus all 即可。

⬇️

3. CUDA Toolkit（容器内）

版本 ≤ 宿主机驱动支持的最高版本。包含 Runtime + 开发工具。不同容器可用不同版本。

⬇️

4. cuDNN / NCCL / 其他库（容器内）

必须与 CUDA Toolkit 版本匹配。通常由框架自动依赖，或打包在 NGC 镜像中。

⬇️

5. PyTorch / TensorFlow（容器内）

框架版本决定所需的 CUDA/cuDNN 版本。如 PyTorch 2.3 需要 CUDA 12.1 + cuDNN 8.9。

🐳 宿主机 vs 容器化安装对比

组件 宿主机安装 容器内安装 说明
NVIDIA Driver ✅ 必须 ❌ 不需要 唯一必须装在宿主机的组件
NVIDIA Container Toolkit ✅ 需要 ❌ 不需要 让 Docker 能访问 GPU
CUDA Toolkit ⚪ 可选 ✅ 推荐 容器化可实现多版本共存
cuDNN ⚪ 可选 ✅ 推荐 与 CUDA 版本严格绑定
PyTorch/TensorFlow ⚪ 可选 ✅ 推荐 框架版本与 CUDA 版本需匹配

🛠️ 快速安装参考

宿主机：安装驱动 + Container Toolkit

# Ubuntu 安装驱动（示例） sudo apt update sudo apt install nvidia-driver-535 # 安装 Container Toolkit sudo apt install nvidia-container-toolkit sudo nvidia-ctk runtime configure --runtime=docker sudo systemctl restart docker # 验证 nvidia-smi docker run --rm --gpus all nvidia/cuda:12.1.0-base-ubuntu22.04 nvidia-smi

容器：使用 NGC 镜像（推荐）

# 拉取预配置好的 PyTorch 环境 docker pull nvcr.io/nvidia/pytorch:26.04-py3 # 运行并挂载代码和数据 docker run --gpus all -it --rm \ -v $(pwd)/project:/workspace \ -v /data:/data \ nvcr.io/nvidia/pytorch:26.04-py3 \ bash # 容器内已包含：CUDA 12.x + cuDNN + PyTorch + NCCL

版本兼容性检查

# 查看驱动支持的最高 CUDA 版本 nvidia-smi # 右上角显示 CUDA Version: 12.4 # 查看 PyTorch 使用的 CUDA 版本 python -c "import torch; print(torch.version.cuda)" # 查看 cuDNN 版本 python -c "import torch; print(torch.backends.cudnn.version())" # 验证 GPU 是否可用 python -c "import torch; print(torch.cuda.is_available())"
